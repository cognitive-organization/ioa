### DEC-2026-04-04-002

- **Date:** 2026-04-04
- **Project:** FINANFIX (paribali-labs/finanfix)
- **Decision:** Replace event-based callback idempotency with an atomic database lock using a conditional UPDATE (`onboarding_steps.current_step 5→6→7`); only the first concurrent callback acquires the lock, all others return early.
- **Context:** The callback was firing 3–5× from 6 callers, causing duplicate categorization runs that exhausted token budgets and corrupted state; event-based idempotency (checking for AccountConnected events within 5 min) had a race condition window between check and insert.
- **Alternatives:** Distributed lock via Redis; database advisory lock; keep event-based guard with shorter window
- **Made by:** Human (Paulo Lima) + Agent (Claude Opus 4.6)
