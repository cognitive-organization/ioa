### DEC-2026-04-02-001

- **Date:** 2026-04-02
- **Project:** FINANFIX (paribali-labs/finanfix)
- **Decision:** Adopt content-based deduplication (account + date + amount + description tuple) as the canonical transaction identity key instead of relying on Pluggy-assigned UUIDs.
- **Context:** Pluggy assigns a new UUID to the same logical transaction on re-sync, so UUID-based upserts were silently creating duplicates; the 7M+ duplicate incident (2026-03-10) was caused by this identity mismatch compounded by a missing UNIQUE constraint from migration 002.
- **Alternatives:** Rely solely on UNIQUE constraint after applying migration 002; deduplicate at display layer rather than storage layer
- **Made by:** Human (Paulo Lima) + Agent (Claude Opus 4.6)
- **Source:** commit 34d079f
- **Review date:** 2026-05-02
